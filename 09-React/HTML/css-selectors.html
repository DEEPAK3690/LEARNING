CSS Parent & Child Selector Examples

Example 1 — Descendant Selector (space)

.ex1-box p { color: blue }

Every <p> at any depth inside .ex1-box turns blue.

.ex1-box

p I am a direct child paragraph → blue ✓

div

p I am a grandchild paragraph → also blue ✓

div

p I am a great-grandchild → still blue ✓

Example 2 — Direct-Child Selector (>)

.ex2-outer > p { color: green }

Only the immediate child <p> changes. Nested ones stay black.

.ex2-outer

p (direct child) I am a direct child → green ✓

.ex2-inner (div)

p (grandchild) I am a grandchild → stays black ✗

Example 3 — Three Levels Deep

.ex3-page .ex3-section .ex3-card h3/p

Read left → right: page → section → card → element.

.ex3-page
.ex3-section

h3 Card Title — styled ✓

p Card description text — styled ✓

h3 Another Card — also styled ✓

p Both cards get the same styles ✓

Example 4 — Mixing Descendant & Direct-Child

.ex4-list span → purple  |  .ex4-item > span → adds background
    .ex4-list
  • .ex4-item Direct child span → purple + background ✓
    div (nested) Grandchild span → purple only, NO background ✓
  • .ex4-item Another direct child span → purple + background ✓

Example 6 — Specificity: Short vs Long Selectors

.ex6-text → red  |  .wrapper .inner .ex6-text → green  |  > > → purple wins

Longer (more specific) selectors override shorter ones, even if written first.

.ex6-wrapper
.ex6-inner

.ex6-text I should be RED (.ex6-text alone), but the longer path overrides me → PURPLE ✓

Example 7 — Horizontal Layout (Flexbox Row)

.ex7-row { display:flex; flex-direction:row }

Parent uses display:flex. Direct child items line up side by side. Each child can be styled independently.

.ex7-item

Item 1

flex child
.ex7-item

Item 2

flex child
.ex7-item

Item 3

.ex7-item

Item 4

flex child

CSS used: .ex7-row { display:flex; gap:12px }  |  .ex7-row > .ex7-item { flex:1 } — each child shares equal width.

Example 8 — Vertical Layout (Flexbox Column)

.ex8-col { display:flex; flex-direction:column }

Same idea as row, but children stack top to bottom. Useful for sidebars, form fields, card bodies.

.ex8-nav (column) 🏠 Home 👤 Profile ⚙️ Settings 📄 Docs
.ex8-form (column)

CSS used: .ex8-col { display:flex; flex-direction:column; gap:8px }  |  .ex8-col > .ex8-nav-item and .ex8-col > .ex8-form-label target direct children only.

Example 9 — Combined: Row of Vertically Stacked Cards

.ex9-row { flex-direction:row }  |  .ex9-card { flex-direction:column }

Parent row is horizontal. Each card child is vertical inside. This is the most common real-world pattern.

Quick Cheat Sheet

Selector Meaning Example
.a b Any b inside .a (any depth) .card p
.a > b Only DIRECT child b of .a .card > p
.a .b .c 3-level nesting: .c inside .b inside .a .app .grid .card
.a .b > c Mix: c is a direct child of .b, anywhere in .a .list .item > span
